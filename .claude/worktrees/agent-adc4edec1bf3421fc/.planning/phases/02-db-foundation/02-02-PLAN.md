    - Import Database from 'better-sqlite3', readFileSync from 'fs', join from 'path'
    - Declare CSV_PATH = join(process.cwd(), 'data', 'worldcities.csv')
    - Declare DB_PATH = join(process.cwd(), 'data', 'cities.db')
    - Read the CSV with readFileSync(CSV_PATH, 'utf-8')

    - Execute DROP TABLE IF EXISTS cities followed by CREATE TABLE cities with schema: id INTEGER PRIMARY KEY, name TEXT NOT NULL, city_ascii TEXT NOT NULL, country TEXT NOT NULL, region TEXT NOT NULL, lat REAL NOT NULL, lon REAL NOT NULL, timezone TEXT NOT NULL DEFAULT '', population INTEGER NOT NULL DEFAULT 0
    - NO CREATE INDEX statements (per D-01)
    - Prepare an INSERT INTO cities statement for all 9 columns using positional ? placeholders
    - Wrap all inserts in a db.transaction() for batch performance
    - For each data line (index 1 to end): skip empty lines; split on comma stripping surrounding double-quotes; extract id=parseInt(fields[col('id')],10), name=fields[col('city')], city_ascii=fields[col('city_ascii')], country=fields[col('country')], region=fields[col('admin_name')], lat=parseFloat(fields[col('lat')]), lon=parseFloat(fields[col('lng')]), timezone='', population=parseInt(fields[col('population')],10)||0; skip rows where id is falsy or name is empty
    - Call insertAll(rows) then db.close()
    - Print: console.log(`Built cities.db: ${rows.length} cities inserted at ${DB_PATH}`)
    - No shebang line (invoked as: node scripts/build-cities-db.mjs)
    - Script is idempotent: DROP TABLE IF EXISTS makes it safe to re-run

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | data/cities.db publicly downloadable | accept (mitigated by placement) | DB is at data/ (project root), not public/ — Next.js does not serve files outside public/ as static assets; verified by D-03 decision |
| T-02-05 | Tampering | build script modifies cities.db on re-run | accept | DROP TABLE IF EXISTS makes it idempotent; intended behavior — devs regenerate from source CSV; Railway never runs the script |
| T-02-06 | Denial of Service | corrupted or truncated cities.db committed | mitigate | Acceptance criteria enforces row count 7000–7500 before commit; fileMustExist: true in db.ts will throw loudly if the DB is corrupt at runtime |
| T-02-07 | Spoofing | external Simplemaps download tampered in transit | accept | Simplemaps Basic is a public MIT-licensed dataset of city names with no PII; integrity compromise would produce wrong city names, not a security incident |
</threat_model>

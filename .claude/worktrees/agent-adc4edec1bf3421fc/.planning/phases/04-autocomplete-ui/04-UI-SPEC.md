## Spacing Scale

Declared values (multiples of 4 only). Source: codebase audit of `SearchBar.tsx` and `WeatherCard.tsx` — project uses px-4/py-2 patterns throughout.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline separators |
| sm | 8px | Input vertical padding (py-2 = 8px), compact element spacing |
| md | 16px | Input horizontal padding (px-4 = 16px), list item padding |
| lg | 24px | Section separators |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Dropdown list items: `px-4 py-2` (16px / 8px) — matches existing `<input>` padding rhythm
- Touch target for `<li role="option">`: minimum 32px height achieved by `py-2 text-sm` (8px + line-height)
- Remove button (`×`): `h-6 w-6` (24px square) — existing WeatherCard pattern; do not change

## Typography

Source: WeatherCard.tsx audit + SearchBar.tsx existing classes. No custom type scale — project uses Tailwind's default scale.

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height |
|------|------|---------------|--------|-------------|-------------|
| Input text | 16px (base) | `text-base` (implicit — no size class on input) | Regular 400 | (default) | 1.5 |
| Suggestion label | 14px | `text-sm` | Regular 400 | (default) | 1.5 |
| City name prefix (matched) | 14px | `text-sm` | Bold 700 | `font-bold` via `<strong>` | 1.5 |
| No-results / error copy | 14px | `text-sm` | Regular 400 | (default) | 1.5 |

Rules:
- Exactly 2 weights in use: 400 (regular) and 700 (bold, only for matched prefix via `<strong>`)
- `<strong>` renders at browser-default bold (700) — no additional class needed
- Do NOT add `font-semibold` (600) to suggestion rows; the existing `font-semibold` usage is in WeatherCard city names (not in dropdown)
- Input placeholder: `placeholder-white/60` (inherits 16px size, weight 400)

## Color

Source: SearchBar.tsx lines 86–113 + globals.css + WeatherCard.tsx. Project is dark-theme with glassmorphism.

| Role | Value / Class | Usage |
|------|--------------|-------|
| Dominant (60%) | `bg-white/20` + `backdrop-blur-sm` | Input field surface; page background is condition-gradient driven |
| Secondary (30%) | `bg-white/90` + `backdrop-blur-sm` | Dropdown panel background (light surface over dark page) |
| Accent (10%) | `bg-blue-50` / `bg-blue-100` | Exact usage: highlighted suggestion row (`bg-blue-100`), hover state on suggestion rows (`hover:bg-blue-50`) |
| Destructive | `text-red-300` | Duplicate-city error message only |

Color contract specifics:

- Input border: `border-white/30` — 1px translucent white, matches existing
- Input focus ring: `focus:ring-2 focus:ring-white/50` — existing; do not change
- Dropdown border: `border-white/20` — existing; keep for panel
- Suggestion text (default): `text-gray-800` — dark text on light dropdown panel
- Suggestion text (no-results / disabled): `text-gray-500` — lighter, communicates inaction
- Highlighted row: `bg-blue-100` — accent, keyboard-navigated item only
- Hover row: `hover:bg-blue-50` — lighter blue; keyboard highlight (`bg-blue-100`) takes precedence
- Full-list disabled suggestion: `opacity-50 cursor-not-allowed` overlay on existing bg
- Duplicate error: `text-red-300` — existing pattern; displayed below input, not inside dropdown
- Match highlight `<strong>` text: inherits `text-gray-800` — no additional color; weight alone differentiates

Accent reserved for:
1. Keyboard-highlighted suggestion row background (`bg-blue-100`)
2. Mouse-hover suggestion row background (`hover:bg-blue-50`)

No other elements may use `bg-blue-*` in this component.

This section supplements standard contract sections for this phase's single-component scope.

### SearchBar container

```
<div ref={containerRef} class="relative w-full max-w-md">
  <input role="combobox" ... />
  <p class="mt-1 text-sm text-red-300">   ← duplicate error (conditional)
  <ul role="listbox" ... >                 ← dropdown (conditional on showDropdown)
    <li role="option" ...>                 ← N result rows OR 1 no-results row
```

### Input element — full attribute contract

| Attribute | Value | Rationale |
|-----------|-------|-----------|
| `type` | `text` | Existing |
| `role` | `combobox` | WAI-ARIA APG combobox pattern |

| `aria-autocomplete` | `list` | Declarative: results appear in a listbox |
| `aria-activedescendant` | `{optionId(highlightedIndex)}` when index ≥ 0, else `undefined` | Points screen reader to highlighted `<li>` |
| `placeholder` | `Search for a city...` | Existing — do not change |
| `className` | `w-full rounded-lg border border-white/30 bg-white/20 px-4 py-2 text-white placeholder-white/60 backdrop-blur-sm focus:outline-none focus:ring-2 focus:ring-white/50` | Existing — do not change |

### Dropdown `<ul>` — full attribute contract

| Attribute | Value |
|-----------|-------|
| `role` | `listbox` |
| `id` | `{listboxId}` — from `useId()` |
| `className` | `absolute z-10 mt-1 w-full rounded-lg border border-white/20 bg-white/90 shadow-lg backdrop-blur-sm` |

### Suggestion `<li>` — full attribute contract

| Attribute | Value |
|-----------|-------|
| `role` | `option` |
| `id` | `{listboxId}-opt-{i}` |
| `aria-selected` | `{i === highlightedIndex}` |
| `tabIndex` | `-1` |
| `onMouseDown` | `(e) => { e.preventDefault(); handleSelect(r) }` — fires before `onBlur`, prevents dropdown close race |
| `className` | `px-4 py-2 text-sm text-gray-800 cursor-pointer` + conditional `bg-blue-100` (highlighted) or `hover:bg-blue-50` (unhighlighted) + `opacity-50 cursor-not-allowed` when `isFull` |

No `<button>` inside `<li>`. ARIA `role="option"` ownership chain must not have a `<button>` child.

| `aria-disabled` | `true` |

## Interaction Contract

### Debounce + AbortController (AUTO-01)

| Property | Value |
|----------|-------|
| Trigger threshold | 2 characters minimum |
| Debounce delay | **200ms** (critical: current code uses 300ms — must be corrected) |
| Cancellation | `AbortController` created inside each `setTimeout` callback; previous controller aborted on every `handleChange` call |
| Error handling | `AbortError` → silent discard; all other errors → silent discard (search is non-critical) |
| Endpoint | `/api/v1/cities/search?q={encodeURIComponent(val)}` |
| On `val.length < 2` | `setResults([])`, `setShowDropdown(false)`, return early |

### Keyboard Navigation (AUTO-03)

| Key | Behaviour | `preventDefault` |
|-----|-----------|-----------------|
| `ArrowDown` | `highlightedIndex = min(i + 1, results.length - 1)` | Yes — prevent page scroll |
| `ArrowUp` | `highlightedIndex = max(i - 1, 0)` | Yes — prevent page scroll |
| `Enter` | If `highlightedIndex >= 0`: call `handleSelect(results[highlightedIndex])`; if index is `-1`: no-op | Yes |
| `Escape` | `setShowDropdown(false)`, `setHighlightedIndex(-1)` | No |
| `Tab` | `setShowDropdown(false)`, `setHighlightedIndex(-1)` — focus moves naturally to next element | **No** — must NOT `preventDefault` |

Initial `highlightedIndex` state: `-1` (nothing pre-selected on open).
Reset `highlightedIndex` to `-1` on every `handleChange` call (before scheduling debounce) to prevent stale index past end of new results.

### Selection behaviour (`handleSelect`)

Existing logic preserved:
1. If `cities.length >= 10`: return (full)
2. If `cities.some((c) => c.id === result.id)`: show duplicate error for 3000ms, return
3. Otherwise: `onAddCity(result)`, `setQuery('')`, `setResults([])`, `setShowDropdown(false)`, `setHighlightedIndex(-1)`

### Dropdown visibility rules

| Condition | `showDropdown` |
|-----------|---------------|
| `query.length < 2` | `false` |
| Fetch returned results (non-empty) | `true` |
| Fetch returned empty results | `true` (to show "No cities found for…" message) |
| User presses Escape / Tab | `false` |
| Click outside container | `false` (existing `document.mousedown` handler — no change needed) |
| City selected | `false` |

## Prefix Highlight Contract (AUTO-04)

Highlight applies to the **city name only**. Region and country are always plain text. The SQLite query matches on `name`, so the prefix is always in the city name portion; highlighting region/country would be misleading.

### `HighlightMatch` helper component

```
function HighlightMatch({ text, query }: { text: string; query: string }) {
  if (!query || !text.toLowerCase().startsWith(query.toLowerCase())) {
    return <>{text}</>
  }
  const prefix = text.slice(0, query.length)
  const rest = text.slice(query.length)
  return <><strong>{prefix}</strong>{rest}</>
}
```

### `formatSuggestion` helper

```
function formatSuggestion(city: CityEntry, query: string): React.ReactNode {
  const suffix = city.region ? `, ${city.region}, ${city.country}` : `, ${city.country}`
  return <><HighlightMatch text={city.name} query={query} />{suffix}</>
}
```

XSS posture: JSX children only — no `dangerouslySetInnerHTML`. React escapes all text nodes.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Screen reader announces dropdown open/closed | `aria-expanded={showDropdown}` on `<input>` |
| Screen reader tracks focused suggestion | `aria-activedescendant={getOptionId(highlightedIndex)}` on `<input>` |
| Screen reader identifies dropdown | `role="listbox"` on `<ul>`, `id={listboxId}` |
| Screen reader identifies each suggestion | `role="option"` on each `<li>`, `aria-selected={i === highlightedIndex}` |
| Screen reader identifies empty state | `role="option" aria-disabled="true"` on no-results `<li>` |
| Focus never leaves `<input>` during keyboard nav | `tabIndex={-1}` on all `<li>` elements; keyboard handled on `<input>` |
| Arrow keys do not scroll page | `e.preventDefault()` in ArrowUp/ArrowDown cases only |
| Tab allows natural focus movement | No `preventDefault` in Tab case |
| Highlight rendering is XSS-safe | JSX `String.slice()` + `<strong>`, never `dangerouslySetInnerHTML` |

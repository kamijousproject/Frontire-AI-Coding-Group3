# Milestone v1.1: Local City Database + Instant Search

**Status:** ✅ SHIPPED 2026-05-13
**Phases:** 02–04
**Total Plans:** 8

## Overview

Replaced live weatherapi.com city search with a bundled SQLite database of 7,300 world cities, enabling instant type-ahead autocomplete and precise lat/lon-based weather lookups. All city resolution now happens server-side against a pre-built `cities.db`, eliminating per-keystroke API calls and improving search reliability.

## Milestone Summary

**Key Decisions:**
- better-sqlite3 (not sql.js WASM) — sync API, no startup overhead, 7K cities is tiny
- `data/cities.db` at project root (not `public/`) — prevents direct browser download
- `process.cwd()` for DB path — `__dirname` breaks in App Router production builds
- `serverExternalPackages` in next.config.ts — prevents webpack from bundling the native binary
- `weather_cities_v2` localStorage key — cannot recover lat/lon from v1.0 string[], silent reset
- Pipe/colon delimiter for `/multiple` — avoids ambiguity with comma in lat,lon pairs
- AbortController created inside setTimeout callback — prevents stale controller from prior keypress
- `onMouseDown` over `onClick` for suggestion selection — fires before `onBlur`

**Issues Resolved:**
- HMR hot-reload creating multiple DB handles — fixed with module-level singleton pattern
- `highlightedIndex` pointing past end of shorter result sets — fixed by resetting to -1 on every `handleChange`
- Debounce was 300ms (pre-existing bug) — corrected to 200ms per AUTO-01 spec

**Issues Deferred:**
- Loading indicator during debounce fetch window (UI-REVIEW finding — cosmetic/UX)
- `font-semibold` weight in WeatherCard (minor typography spec violation)
- Input `aria-label` missing (accessibility gap — screen reader names)
- M1-M8 manual browser tests (require live WEATHER_API_KEY + Railway deployment)
- Production deployment with real WEATHER_API_KEY

**Technical Debt Incurred:**
- No loading spinner during 200ms debounce fetch — users on slow connections see no feedback
- WeatherCard uses `font-semibold` (600) which violates the 2-weight spec (400/700 only)

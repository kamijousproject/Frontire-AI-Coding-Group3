## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-16 | Tampering | SearchBar casting /cities/search response as CityEntry[] without client-side validation | accept | Server-side: data comes from SQLite (trusted DB), filtered through parameterized query; response is validated at DB layer; adding client-side re-validation would be redundant for an internal API; if the API were external, add Zod parse here |
| T-03-17 | Spoofing | XSS via city name in CityEntry.name rendered in SearchBar dropdown | mitigate | React JSX renders text nodes via {r.name} — React escapes HTML entities automatically; no dangerouslySetInnerHTML used; city names from SQLite cities.db are trusted static data |
| T-03-18 | Denial of Service | retryExhausted Map growing unbounded | accept | Bounded by MAX_CITIES (10) — at most 10 entries in retryExhausted at any time; keys are coordinate strings; Map is reset on page reload |
| T-03-19 | Tampering | removeCity passing wrong type (string vs number) causing silent no-op | mitigate | WeatherGrid.onRemoveCity declared as (cityId: number) — TypeScript enforces number type at compile time; onRemoveCity(city.id) where city.id: number is type-safe |
</threat_model>

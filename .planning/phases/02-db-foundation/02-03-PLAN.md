## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Denial of Service | DB connection leak via HMR in dev mode | mitigate | global.__db singleton pattern prevents new connections from opening on each HMR cycle; each process lifetime has exactly one Database instance |
| T-02-09 | Tampering | Write operations through the read-only singleton | mitigate | Database opened with { readonly: true } — any attempted write via getDb() throws immediately with a SQLite error; callers cannot corrupt the DB through this handle |
| T-02-10 | Spoofing | Wrong DB path via __dirname in compiled output | mitigate | Enforced by acceptance criterion: src/lib/db.ts must NOT contain __dirname; process.cwd() always resolves to project root in both dev and Railway |
| T-02-11 | Elevation of Privilege | SQL injection via Phase 03 search query | transfer | Phase 02 scope is the singleton only; Phase 03 is responsible for using db.prepare() with positional parameters (never string interpolation in SQL). Document this expectation in Phase 03 planning. |

<task id="T4" type="auto">
  <title>Rate Limiting Middleware</title>
  <files>src/lib/rate-limit.ts</files>
  <action>
    Create src/lib/rate-limit.ts:

    In-memory store: Map<string, { count: number; resetAt: number }>
    WINDOW_MS = 60_000 (1 minute)
    MAX_REQUESTS = 60

    getClientIp(request: Request): string
    - Read X-Forwarded-For header
    - Split by comma, pop last value (Railway appends real client IP to end — first value is spoofable)
    - Trim result
    - Fallback to '127.0.0.1' if header absent

    checkRateLimit(request: Request): { allowed: boolean; remaining: number }
    - Get IP via getClientIp
    - If no entry for IP, or entry.resetAt < Date.now(): create new entry { count: 1, resetAt: Date.now() + WINDOW_MS }, return { allowed: true, remaining: 59 }
    - Else increment count
    - If count > MAX_REQUESTS: return { allowed: false, remaining: 0 }
    - Return { allowed: true, remaining: MAX_REQUESTS - count }
  </action>
  <verify>
    `grep -n "split" src/lib/rate-limit.ts` shows comma split.
    `grep -n "pop\|slice(-1)" src/lib/rate-limit.ts` shows last-value extraction.
    `npx tsc --noEmit` exits 0.
  </verify>
  <done>checkRateLimit exported. Last X-Forwarded-For value used. TypeScript check passes.</done>
</task>

<task id="T6" type="auto">
  <title>weatherapi.com Client</title>
  <files>src/lib/weatherapi.ts</files>
  <action>
    Create src/lib/weatherapi.ts:

    BASE_URL = 'https://api.weatherapi.com/v1'
    TIMEOUT_MS = 5000

    getApiKey(): string
    - Read process.env.WEATHER_API_KEY
    - If undefined or empty: throw new Error('WEATHER_API_KEY env var is not set')

    normalizeIconUrl(url: string): string
    - If starts with '//': return 'https:' + url
    - Return url unchanged

    fetchCurrentWeather(city: string): Promise<WeatherData>
    - Create AbortController; setTimeout → controller.abort() after TIMEOUT_MS
    - fetch(`${BASE_URL}/current.json?key=${getApiKey()}&q=${encodeURIComponent(city)}&aqi=no`, { signal })
    - If response.status === 400: throw error with code 1001 (city not found via weatherapi)
    - If response.status === 403: throw error with code 1002 (rate limit)
    - If !response.ok: throw error with code 1003
    - If AbortError: throw error with code 1003
    - Parse JSON → map to WeatherData:
        type: 'data'
        city: json.location.name
        country: json.location.country
        region: json.location.region
        temp_c: json.current.temp_c
        temp_f: json.current.temp_f
        feelslike_c: json.current.feelslike_c
        feelslike_f: json.current.feelslike_f
        humidity: json.current.humidity
        wind_kph: json.current.wind_kph
        condition: json.current.condition.text
        condition_code: json.current.condition.code
        icon_url: normalizeIconUrl(json.current.condition.icon)
        is_day: json.current.is_day
        last_updated: new Date().toISOString()

    fetchCitySearch(q: string): Promise<CitySearchResult[]>
    - fetch(`${BASE_URL}/search.json?key=${getApiKey()}&q=${encodeURIComponent(q)}`)
    - If !response.ok: return []
    - Parse JSON array → map to CitySearchResult: name, region, country
    - Return first 5 results (slice(0, 5))
  </action>
  <verify>
    `grep -rn "NEXT_PUBLIC_" src/lib/weatherapi.ts` returns no matches.

    `grep -n "5000\|TIMEOUT_MS" src/lib/weatherapi.ts` confirms 5s timeout.
    `npx tsc --noEmit` exits 0.
  </verify>
  <done>fetchCurrentWeather and fetchCitySearch exported. No NEXT_PUBLIC_ usage. Icon normalization applied. 5s timeout via AbortController. TypeScript check passes.</done>

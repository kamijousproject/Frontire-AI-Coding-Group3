  <action>
    Create src/lib/condition-backgrounds.ts (do not inline in components).

    Define CONDITION_GROUPS mapping condition_code ranges to day/night Tailwind gradient class pairs,
    following the 8-group table in SPEC.md §Frontend Style:
    - 1000: sunny/clear → day: 'bg-gradient-to-b from-sky-400 to-blue-500', night: 'bg-gradient-to-b from-blue-900 to-indigo-950'
    - 1003: partly cloudy → day: 'bg-gradient-to-b from-slate-300 to-blue-300', night: 'bg-gradient-to-b from-blue-900 to-slate-800'
    - [1006,1007,1008,1009]: cloudy/overcast → day: 'bg-gradient-to-b from-gray-400 to-gray-500', night: 'bg-gradient-to-b from-gray-800 to-gray-900'
    - rain codes [1063,1150..1201,1240..1246]: → day: 'bg-gradient-to-b from-blue-700 to-blue-800', night: 'bg-gradient-to-b from-blue-900 to-blue-950'
    - snow codes [1066,1114..1117,1210..1237,1255..1264]: → day: 'bg-gradient-to-b from-blue-100 to-slate-200', night: 'bg-gradient-to-b from-slate-700 to-blue-900'
    - thunder codes [1087,1273..1282]: → day: 'bg-gradient-to-b from-purple-800 to-purple-900', night: 'bg-gradient-to-b from-purple-950 to-gray-950'
    - fog codes [1030,1135,1147]: → day: 'bg-gradient-to-b from-gray-400 to-gray-500', night: 'bg-gradient-to-b from-gray-700 to-gray-800'
    - fallback: → day: 'bg-gradient-to-b from-gray-400 to-gray-500', night: 'bg-gradient-to-b from-gray-700 to-gray-800'

    getConditionGroup(code: number): { day: string; night: string }

    getPageBgClass(code: number, isDay: number): string
    - return getConditionGroup(code)[isDay === 1 ? 'day' : 'night']

    getCardBgClass(code: number, isDay: number): string
    - Same lookup but lighter opacity for card tint; append '/20' or use a secondary class set
    - Return a subtle bg class (e.g. 'bg-white/10' layered over card — or define separate card-specific lighter classes)

    Export: getPageBgClass, getCardBgClass
  </action>

  <action>
    Create src/hooks/useLocalStorage.ts as a React hook: useCityStorage()

    STORAGE_KEY = 'weather_cities'
    MAX_CITIES = 10

    On mount (useEffect):
    - Read localStorage.getItem(STORAGE_KEY)
    - Try JSON.parse; if throws or result is not string[]: set state to [] silently (no console.error)
    - If valid string[]: filter empty strings, lowercase+trim each, slice first 10
    - Set state to normalized array

    addCity(name: string): void
    - Normalize: name.toLowerCase().trim()
    - If already in cities (case-insensitive): do nothing (caller handles duplicate UI)
    - If cities.length >= 10: do nothing
    - Append to state; write updated array to localStorage

    removeCity(name: string): void
    - Filter out city by normalized name (case-insensitive)
    - Write updated array to localStorage

    Return: { cities: string[], addCity, removeCity }

    Note: cities stored as lowercased trimmed strings. Display name shown in cards comes from WeatherData.city (raw from weatherapi.com).
  </action>

<task id="T13" type="auto">
  <title>Weather Data Hook</title>
  <files>src/hooks/useWeather.ts</files>
  <action>
    Create src/hooks/useWeather.ts: useWeather(cities: string[])

    State: results Map<string, WeatherData | ErrorResponse>
    State: loading boolean
    State: retryCount Map<string, number> — tracks per-city retry attempts

    On cities change (useEffect, dep [cities.join(',')]):
    - If cities.length === 0: clear results, return
    - Set loading = true
    - Fetch GET /api/v1/weather/multiple?cities=${cities.join(',')}
    - On success: parse MultiWeatherResult[]; update results map — WeatherData entries by city name (lowercased), WeatherError entries stored as ErrorResponse shape
    - On network error: set all cities to ErrorResponse code 1003
    - Set loading = false

    retryCity(city: string): void
    - Check retryCount.get(city) ?? 0; if >= 1: do nothing (already exhausted retry)
    - Increment retryCount for city
    - Fetch GET /api/v1/weather/current?city=${city}
    - On success: update results for city with WeatherData
    - On failure: update results for city with ErrorResponse (type: 'error') — retryCount already at 1, so caller WeatherCard shows permanent fail state

    Return: { results, loading, retryCity }
  </action>
  <verify>
    Manual: add London → results map contains WeatherData for london.
    Manual: with network mocked to fail → results map contains ErrorResponse for each city.
    Manual: click retry once → retryCity called; click retry again → no action (retryCount >= 1, WeatherCard shows permanent fail message).
    `npx tsc --noEmit` exits 0.
  </verify>
  <done>useWeather hook exported. Fetches multiple cities. retryCity bounded at 1 attempt. TypeScript check passes.</done>
</task>

<task id="T14" type="auto">
  <title>SearchBar Component</title>
  <files>src/components/SearchBar.tsx</files>
  <action>
    Create src/components/SearchBar.tsx.

    Props:
    - cities: string[] (current city list — for duplicate and full-list checks)
    - onAddCity: (city: string) => void

    State:
    - query: string (controlled input)
    - results: CitySearchResult[]
    - showDropdown: boolean
    - duplicateError: boolean (shows "City already added" inline)

    Behavior:
    - Input onChange: update query; if query.length >= 2, schedule debounced fetch (300ms) to GET /api/v1/cities/search?q={query}; if query.length < 2, clear results and hide dropdown
    - Debounce: clear previous timeout on each keystroke; fire after 300ms idle
    - duplicateError: clear on next keypress (set to false in onChange before debounce)
    - On search response: set results, showDropdown = results.length > 0
    - Outside click: attach document mousedown listener on mount, remove on unmount; if click outside component ref → setShowDropdown(false)
    - Escape key: onKeyDown on input → if key === 'Escape': setShowDropdown(false)

    Dropdown:
    - Each result: `{result.name}, {result.region}, {result.country}`
    - If cities.length >= 10: each result rendered as disabled button with title="City list full (10/10)"
    - Else: clickable button; onClick: check duplicate (cities.includes(result.name.toLowerCase().trim())); if duplicate: setDuplicateError(true), setTimeout clear after 3000ms; else: onAddCity(result.name), clear query, hide dropdown

    Below input (conditional): if duplicateError → `<p>City already added</p>` in red text

    Empty search state: if results.length === 0 and showDropdown → show "No cities found" in dropdown
  </action>
  <verify>
    Manual: type "Lon" → dropdown appears after 300ms with up to 5 results showing city, region, country.
    Manual: type "Lon", click outside → dropdown closes.
    Manual: type "Lon", press Escape → dropdown closes.
    Manual: add London, search "Lon", click London again → "City already added" appears below input, vanishes after 3s.
    Manual: with 10 cities added, open dropdown → each result is disabled with "City list full (10/10)" tooltip.
    `npx tsc --noEmit` exits 0.
  </verify>
  <done>SearchBar renders with debounced search, dropdown, duplicate error, and 10-city disabled state. TypeScript check passes.</done>
